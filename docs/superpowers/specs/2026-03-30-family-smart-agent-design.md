## 1. 产品定位

**一句话：** 小艺只懂「个人」，我做的是懂「家庭」的 AI 协同大脑。

**核心差异：** 小艺/小爱只能执行单次指令，无法记住家庭成员特征、无法做跨场景协同。家庭智联事务中枢通过「家庭记忆图谱 + 自动学习引擎」实现真正的上下文理解和智能任务分配。

## 2. 核心理念升级（借鉴 Floatboat）

### 2.1 从"存储"到"学习"

| 旧思路 | 新思路（Floatboat 风格）|
|--------|----------------------|
| 用户说"妈妈腰不好" → 记住 | 用户完成几次任务 → AI 自动蒸馏出"家庭规则" |
| 被动存储记忆 | 主动学习模式 |
| 每次都需要用户说清楚 | AI 观察行为，主动补全上下文 |

### 2.2 Combo 思想 → 家庭场景模板

Floatboat 的 Combo = 工作流打包成可复用模板。

我们的对应概念：

| Floatboat Combo | 我们的"家庭场景模板" |
|----------------|---------------------|
| "每周做报告"是一个 Combo | "野餐"是一个场景模板 |
| 自动记住工作步骤 | 自动记住参与成员、默认设备、典型时间 |
| 下次一键启动 | 下次说"野餐"，AI 自动填充上下文 |

### 2.3 自我进化能力

> "当 AI 能感知你 80% 的操作时，它就有自我进化的能力了" — Floatboat

我们的实现：
- 记录任务分配模式，发现规律
- 比如："每次安排爸爸的任务，他喜欢下午做" → 蒸馏成家庭规则
- AI 主动推荐："按以往习惯，我把这件事安排在下午3点可以吗？"

## 3. MVP 三大核心场景

### 场景一：复杂自然意图理解（升级版）

**用户说：** "下周三代家人去公园野餐，帮我列采购清单，提醒老公准备帐篷"

**AI 理解过程：**
1. 识别这是一个「家庭野餐」事件
2. **检查是否有相关场景模板**（如果之前有野餐记录，直接复用）
3. 拆解出多个子任务：采购清单创建、提醒设置、设备同步
4. 识别家庭成员角色：老公（准备帐篷）
5. **Combo 自动推荐**："要不要把这次野餐存为一个场景模板？"

### 场景二：家庭级永久记忆 + 自动学习（核心升级）

**用户说：** "妈妈最近腰不好，帮我安排每周两次的家务提醒，别让她提重物"

**AI 记忆过程：**
1. 提取「妈妈」这个角色，存入家庭成员图谱
2. 记录属性：腰不好（健康特征）、需避免提重物（照料约束）
3. **主动学习：** 后续安排家务时，自动规避分配给妈妈的重物任务
4. **蒸馏规则：** 如果用户多次手动调整了分配方案，AI 学习规律并主动应用

**效果：** 家庭记忆越丰富，AI 越懂这个家庭，后续无需重复说明。AI 越用越聪明。

### 场景三：鸿蒙全场景协同推送 + IACT 风格内联操作

**任务生成后，展示 IACT 风格的内联操作：**

```
📋 采购清单 → [推送到手机] [同步智慧屏] [分享给家人]
📋 提醒老公准备帐篷 → [现在提醒] [稍后提醒] [添加到日历]
```

用户不需要跳转到设备页，直接在对话卡片里一键操作。

## 5. 家庭记忆图谱设计（升级版）

### 数据模型

```typescript
// 家庭成员
interface FamilyMember {
  id: string;
  name: string;           // 称呼：如"妈妈"
  relation: string;       // 关系：如"母亲"
  healthNotes: string[];  // 健康备注：如["腰不好"]
  habits: string[];       // 习惯偏好：如["下午有空"]
  constraints: string[];  // 照料约束：如["避免提重物"]
  learnedRules: string[]; // 【新增】从行为中学习的规则：如["喜欢下午做事"]
  lastMentioned: number;
}

// 家庭场景模板（Combo 对应概念）
interface FamilyScene {
  id: string;
  name: string;           // 如"野餐"
  description: string;    // 场景描述
  participants: string[];  // 参与成员
  defaultDevices: string[];  // 默认同步设备
  typicalTime?: string;     // 典型时间
  isCombo: boolean;         // 【新增】是否已"蒸馏"为Combo
  usageCount: number;       // 【新增】使用次数（越多越优先推荐）
  createdAt: number;
}

// 记忆任务
interface MemoryTask {
  id: string;
  content: string;
  summary: string;           // AI 提取的摘要
  sourceMemberId: string;
  relatedMembers: string[];
  sceneId?: string;
  remembered: boolean;
  // 【新增】自我进化相关
  isLearnedRule: boolean;   // 是否已蒸馏为规则
  relatedTaskPattern?: string; // 关联的任务模式
}

// 【新增】家庭学习规则
interface FamilyLearnedRule {
  id: string;
  type: 'timing' | 'assignment' | 'preference' | 'constraint';
  pattern: string;         // "爸爸的任务安排在下午"
  confidence: number;      // 置信度（使用越多越高）
  sourceTasks: string[];   // 来源任务ID
  autoApply: boolean;     // 是否自动应用
}
```

### 存储方式
- 使用鸿蒙分布式数据管理
- 新增 `FamilyLearnedRule` 表存储学习规则
- 场景模板 `isCombo=true` 时优先级更高

## 7. 评委演示方案（升级版）

**演示流程（控制在 3 分钟内）：**

1. **开场（10秒）：** "我叫家庭智联事务中枢，和小艺不一样——小艺每次都要你从头说，我懂整个家庭，越用越懂你。"

2. **场景演示（2分钟）：**
   - 第一次："下周三代家人去公园野餐，帮我列采购清单，提醒老公准备帐篷"
   - AI 生成任务，用户确认推送
   - **Combo 推荐：** "要把这次野餐存为场景模板吗？下次说'野餐'就能一键启动"
   
   - 第二次："妈妈腰不好，帮我安排家务"
   - **自动规避：** AI 知道妈妈腰不好，不分配重物
   - **主动学习展示：** "我注意到您多次把重物任务分配给爸爸，我记住了这个习惯，下次自动安排？"

3. **差异化总结（30秒）：**
   - "小艺只执行命令，我们记住家庭、自动学习"
   - "这就是 Floatboat 说的'工作环境'而非'助手'"

## 8. 开发优先级（升级版）

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | 小艺入口 + 基础对话 | 入口必须通 |
| P0 | 意图理解 + 拆解 | 核心能力 |
| P0 | 家庭记忆图谱 | 差异化壁垒 |
| P0 | 任务生成 + 推送 + IACT内联按钮 | 评委可演示 |
| P1 | **场景模板（Combo）系统** | Floatboat 核心差异化 |
| P1 | 设备选择 UI | 协同能力展示 |
| P2 | **家庭学习引擎（规则蒸馏）** | 自我进化，长期价值 |

## 9. 技术风险（同前）

| 风险 | 影响 | 缓解方案 |
|------|------|---------|
| 小艺 MCP 调试环境缺失 | 无法本地测试 | 使用华为云调试环境 |
| Gemini API 响应延迟 | 体验不流畅 | 本地缓存 + 流式输出 |
| 分布式数据同步冲突 | 多设备数据不一致 | 使用鸿蒙官方分布式事务 |
| 决赛现场网络不稳定 | 云端能力失效 | 核心流程支持弱网降级 |

## 10. 与 Floatboat 的对比说明（评委问题预判）

| 维度 | Floatboat | 我们的产品 |
|------|----------|------------|
| 定位 | 通用 AI 工作环境 | **家庭垂直场景** |

| 优势 | 通用性强 | **场景深耕，家庭需求更刚性** |
