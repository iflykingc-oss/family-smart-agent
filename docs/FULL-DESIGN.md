## 一、产品定位

### 1.1 一句话定位

**家庭版的"第二大脑"** —— 帮整个家庭记住每个成员的需求、偏好、健康状况，越用越懂这个家庭。

### 1.2 核心定位

**全生活化・鸿蒙专属家庭认知管家**

渗透"柴米油盐、健康、育儿、家务、人情、习惯"的全场景生活化智能。

### 1.3 产品愿景

```
不是冷冰冰的工具，是懂家人的温暖存在

小艺只会"收到指令，执行命令"
我们是"你随口一说，我默默记住，关键时刻主动提醒"
```

## 二、用户故事

### 场景1：随口叮嘱，AI 记住
```
用户说："妈妈最近腰不好，别让她提重东西"
    ↓
AI 温柔回应："记住了，妈妈腰不好，重活我来帮您安排给其他人"
    ↓
写入家庭知识图谱 → 以后安排家务自动规避
```

### 场景2：反复出现，自动养成规则
```
第1次："爸爸周三下午要开会" → "好的，已设置提醒"
第2次："爸爸周三下午要开会" → "这次也记下了，看来是常事呢"
第3次："爸爸周三下午要开会" → "爸爸周三开会已成为习惯啦，以后每到周三我会提前提醒哦"
```

### 场景3：定时推送，贴心提醒
```
任务："提醒老公开会"
    ↓
定时：周三下午2:45
    ↓
推送文案："老公，明天3点要开会啦，记得带好资料哦～"
    ↓
多设备：手机通知 + 智慧屏弹窗 + 音箱语音
```

## 三、核心能力矩阵

| 能力 | 说明 | 用户感知 |
|------|------|---------|
| **随口叮嘱 → 记住** | 任何生活细节随口说，AI温柔确认并记住 | "不用反复说" |
| **临时记忆 → 长效规则** | 反复出现的提醒，自动升级为长效规则 | "越用越懂我们家人" |
| **单设备 → 全家协同** | 任务推送至全家所有设备 | "每个人都能看到" |
| **被动响应 → 主动预见** | AI主动推荐、主动提醒、主动关怀 | "比家人还贴心" |
| **定时推送 → 温暖送达** | 准时提醒 + 贴心文案 + 对应的人 | "它比我还上心" |

## 四、差异化壁垒

### 4.1 对比小艺/小爱

| | 小艺/小爱 | 我们的产品 |
|--|----------|------------|
| **记忆范围** | 单用户 | **全家庭** |
| **记忆时效** | 单次会话 | **永久 + 自动进化** |
| **任务协同** | 单设备 | **全家多设备** |
| **理解深度** | 命令执行 | **懂家庭成员特点** |
| **交互温度** | 机械回应 | **温暖、贴心、人情味** |

### 4.2 技术壁垒

1. **家庭知识图谱** — 三层记忆（事实/模式/历史），跨任务理解
2. **约束推理引擎** — 健康约束自动传导（腰不好≠提重物）
3. **模式学习器** — 临时规则自动进化为长效规则
4. **场景引擎** — 全生活场景可叠加
5. **温情文案生成** — 推送内容有人情味

```
┌──────────────────────────────────────┐

                   ↓

│  用途：回顾、关联、模板生成         │
└──────────────────────────────────────┘
```

## 六、数据模型

### 6.1 语义记忆
```typescript
interface SemanticMemory {
  id: string;
  type: 'health' | 'preference' | 'constraint' | 'relation';
  memberId: string;
  content: string;      // "妈妈腰不好"
  summary: string;       // "腰不好"
  keywords: string[];
  confidence: number;   // 置信度
  usageCount: number;    // 被调用次数
  createdAt: number;
}
```

### 6.2 程序记忆（行为模式）
```typescript
interface ProceduralMemory {
  id: string;
  pattern: string;      // "爸爸的任务安排在下午"
  trigger: string[];
  action: string;
  confidence: number;
  evidence: string[];     // 来源任务ID
  autoApply: boolean;     // 自动应用
}
```

### 6.3 约束
```typescript
interface Constraint {
  memberId: string;
  condition: string;       // "腰不好"
  forbiddenActions: string[];  // ["提重物", "搬重东西"]
}
```

### 6.4 家庭场景（Combo）
```typescript
interface FamilyScene {
  id: string;
  name: string;           // "野餐"
  participants: string[];
  typicalTasks: TaskTemplate[];
  defaultDevices: string[];
  usageCount: number;
  autoRecommendThreshold: number;
}
```

## 七、评委演示方案

### 7.1 演示流程（3分钟）

**开场（10秒）：**
> "我叫家庭智联事务中枢，和小艺不一样——小艺每次都要你从头说，我懂整个家庭，越用越懂你。"

**场景演示（2分钟）：**

**第一幕：随口记住**
- 用户说："妈妈腰不好，帮我安排家务"
- AI 回应："记住了，妈妈腰不好，重活我来帮您安排"
- 展示：记忆被写入知识图谱

**第二幕：自动规避**
- 用户说："安排搬运物资"
- AI 自动识别：妈妈有腰伤约束 → 分配给爸爸
- 展示：约束推理结果

**第三幕：定时推送**
- 展示：任务推送卡片
- 展示：温情文案风格

**结尾（30秒）：**
> "这就是家庭第二大脑——你随口一说，我记住；关键时候，我提醒。"

### 7.2 差异化总结

```
小艺 = 单次记忆，机械回应
我们 = 永久记忆，温暖提醒，越用越懂
```

## 八、推送文案风格

### 8.1 冷漠 vs 温暖

| 场景 | 冷漠版 | 温暖版 |
|------|--------|--------|
| 老公开会 | "提醒：下午3点开会" | "老公，明天3点要开会啦，别忘了哦～" |
| 妈妈复查 | "提醒：明天10点复查" | "妈妈的复查是明天上午10点，我已经帮您约好车了" |
| 买牛奶 | "提醒：购买牛奶" | "老公，今天记得带牛奶回来，家里快没啦" |

### 8.2 文案生成规则
- 称呼亲人（老公/妈妈/儿子）
- 加语气词（啦/哦/呢）
- 加关心（别忘了/记得/放心）
- 特殊日子主动延伸
